整套家装业务的核心逻辑可以概括为：**以 “人 + 材料” 为底层建模，通过 “分类→分项→定额” 的三级封装把细碎的家装商品与工艺转化为客户可感知的报价项；再以「主材任务」为主轴，串联报价、选品、复尺、确品、下单全流程；后端通过供应链 “接备发”+ 施工包模式完成履约交付，最终按工地进度分批验收**。

## **主流程链路层**—— 从前到后串联一条主材任务从报价到交付的完整流转路径
### 逐段业务解释

1. **主轴：主材任务**
    
    整条链路以「主材任务」为核心载体，**一个主材任务对应一个材料品类**。一套房子的整体交付，由多条并行的主材任务共同组成。任务轴的核心作用是 “管人、管履约进度”，工地数量少时可人工盯控，规模上来后必须靠系统以任务为单位做管理。
    
2. **报价**
    
    任务起点先出报价，采用宽泛的分项定额模式，目的是快速和客户达成一致、完成收款，避免因过度细化导致客户流失。
    
3. **选品**
    
    客户在报价框架内选定品类档次（例如选定 “高端门” 档位）。此时只确定大类和档次，不锁定具体型号、精确尺寸，属于 “粗选型”。
    
4. **复尺 → 变更**
    
    选品后到现场复尺，获取精确的长宽高等尺寸数据；尺寸变化会带来规格、价格变动，触发**变更**流程，变更结果同步回写报价单。
    
    复尺 + 变更是家装的典型逆向调整环节，也是 “宽泛签约、后期落地” 模式的必然产物。
    
5. **确品**
    
    复尺变更完成后，锁定最终商品：品牌、型号、尺寸、价格全部确定。纪要明确提到：确品和下单基本同步，可视为同一个阶段。
    
6. **下单 → 供应链_接备发**
    
    确品完成后正式下单，订单进入供应链体系，触发第二张图的 “接备发→施工包→验收” 履约流程。
    
    整条链路的核心驱动逻辑是**工地进度**：发货、安装都要匹配工地施工阶段，不是一次性全部发货、全部安装。

## **报价模型层**—— 解决 “家装细碎、难快速签单” 的定价与产品封装问题

家装业务技术模型、报价方式演变、定额与分项概念、分类 - 分项 - 工艺商品三层关系。

### 逐段业务解释

1. **底层拆解：装修 = 人 + 材料**
    
    家装整体遵循 “人料机” 模型，核心落地为「人（人工 / 工种）」和「材料（主辅材）」两大维度，其余为辅助因素。这是整个报价体系的最小单元。
    
2. **中间层：分类**
    
    对人和材料做一级归类，是商品与工种的目录：
    
    - 材料侧：按商品品类划分，如门、窗、地板等；
    - 人工侧：按工种划分，对应木工、瓦工、水电工等。
        
        分类的作用是把海量 SKU 和工种做收纳，让后续报价可检索、可配置。
    
3. **核心层：分项（选品分项 / 材料分项 / 人工分项）**
    
    分项是介于 “分类” 和 “具体商品 / 工艺” 之间的中间封装层，也是客户能直接感知、选择的单元：
    
    - **选品分项**：对应主材，给客户提供档次化选项（如 “高端门”“低端门”），是 “宽泛报价、快速签单” 的核心载体；
    - **材料分项**：对应辅材，一般不直接面向客户展示；
    - **人工分项**：对应工种的施工服务。
        
        纪要中提到，最初报价做太细导致签单慢、回款慢；现行方案就是用 “分项 + 档次” 的宽泛模式（如直接选高端门）让客户快速决策签合同，后期再落地细节、多退少补。
    
4. **输出 1：生成报价单 → 生成定额给客户**
    
    选定分项后（例如 “高端门 × 1，1000 元”），系统生成报价单；最终面向客户呈现的不是零散的材料 + 人工明细，而是**定额**—— 即主辅材 + 人工打包后的成品项目。定额价格由每一项底层成本反推得出，客户只需要对定额项付费，不用感知细碎的组成。
    
5. **输出 2：确品 → 匹配具体品牌与尺寸**
    
    宽泛的 “高端门” 只是签约阶段的选型；签约后需要落地为具体商品：明确品牌、具体型号、精确尺寸。确品是 “从宽泛报价到精确履约” 的衔接点，和复尺、变更强关联。

## **履约交付层**—— 解决 “材料发货、工人安装、验收” 的后端落地问题
供应链接备发流程、施工包与班组管理、三种工人模式、分批 / 批量验收、工地进度协同。

### 逐段业务解释

1. **起点：接备发（供应链三大动作）**
    
    订单下发供应商后，履约从 “接单 - 备货 - 发货” 开始：
    
    - **供应链接单**：供应商接收订单，进入生产 / 备货队列；
    - **备货**：标准品直接备货，定制类商品（如定制柜）涉及排产，周期更长；
    - **发货**：分为「单发」和「齐发」两种模式。
        
        - 单发：单品类按需单独发货；
        - 齐发：多品类集中一次发货。
            
            发货节奏**完全跟随工地进度**，提前预约送货日期，避免材料提前进场损坏、或已安装部位被后续施工破坏。
        
    
2. **安装单元：施工包**
    
    材料进场后，安装不以单个工人为单位，而是以**施工包**为单位组织作业（例如水电包、瓦工包）。施工包同时包含「人」和「料」两部分，是结算、管理的最小单元。
    
3. **人员构成：三类工人模式**
    
    施工包里的 “人” 分为三种模式，各有优劣：
    
    - **工长模式**：传统包工头带队，灵活、管理成本低，但工艺标准不统一；
    - **自营工人**：公司直招，可统一培训工艺、保障安装质量，但人力与管理成本高；
    - **供应商工人**：购买商品时附带安装服务，由供应商负责工人。
        
        纪要补充：工人与工地是**多对多**关系 —— 一个班组同时跑多个工地，一个工地有多个班组先后进场，这也是家装履约细碎、管理难度高的原因之一。
    
4. **终点：验收**
    
    施工包完成后进入验收环节，遵循 “分批验收 + 批量验收结合” 的原则：
    
    - 隐蔽工程、易被后续工序覆盖的项目，装完立即验收，避免后续无法核查；
    - 可统一验收的项目批量验收，控制监理 / 验收人员的上门成本。
        
        验收通常由监理或第三方验收公司执行，是保障交付质量、提升业主体验的关键节点。
